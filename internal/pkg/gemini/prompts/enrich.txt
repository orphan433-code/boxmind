Category rules:

Classification algorithm (FOLLOW IN THIS ORDER — main rule):

Step 1 — What does the USER DO on this page? Category = action/format, NOT topic.
  WATCH video (film, series, anime, vlog, documentary, interview, clip) -> movies or entertainment
  STUDY / learn (course, lesson, tutorial, lecture, how-to, sheet music to practice) -> learning
  READ text (wiki, blog, longread, reference) -> articles
  READ news article (text on a news site) -> news
  LISTEN to music (track, album, playlist) -> music
  USE while coding (official docs, API reference, package readme) -> programming
  PLAY or read about games -> gaming
  SHOP (product listing, store) -> shopping

Step 2 — What is it ABOUT? Topic goes in tag #2 only — never pick category from topic alone.

Common mistakes to avoid:
  WRONG: politics / war / current events topic -> news (when user WATCHES a video)
  RIGHT: watch political video -> entertainment, tags: ["видео", "политика"]

  WRONG: golang / react / python topic -> programming (when user STUDIES via course or tutorial)
  RIGHT: YouTube golang course -> learning, tags: ["курс", "golang"]

  WRONG: film or anime topic -> articles (when user WATCHES on a streaming page)
  RIGHT: watch anime -> movies, tags: ["аниме", "фэнтези"]

Same platform — different categories (decide by Step 1, not by domain):
